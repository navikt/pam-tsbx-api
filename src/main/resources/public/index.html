Token sandbox API ⏳️

User messages can be fetched from API available at /api/v1/messages.

It requires a bearer access token in Authorization header with a token obtained by exchanging id-porten identity token using a token exchange service. The token audience must be pam-tsbx-api. The token should at minimum contain the claims pid (Norwegian NIN) and name, so that this API can identify which end user the API calls are made on behalf of.
